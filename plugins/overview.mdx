## Plugin Manager

nvim-luffy uses [lazy.nvim](https://github.com/folke/lazy.nvim) for efficient plugin management. It provides lazy-loading capabilities, automatic dependency resolution, and a clean plugin configuration structure.

### Setup

The plugin manager is configured in `lua/leyland/lazy.lua:1-22`:

```lua
local lazypath = vim.fn.stdpath("data") .. "/lazy/lazy.nvim"
if not vim.loop.fs_stat(lazypath) then
  vim.fn.system({
    "git",
    "clone",
    "--filter=blob:none",
    "https://github.com/folke/lazy.nvim.git",
    "--branch=stable",
    lazypath,
  })
end
vim.opt.rtp:prepend(lazypath)

require("lazy").setup(
  { 
    { import = "leyland.plugins" }, 
    { import = "leyland.plugins.lsp" } 
  },
  {
    checker = {
      enabled = true,
      notify = false,
    },
    change_detection = {
      notify = false,
    },
  }
)
```

### Plugin Organization

Plugins are organized into the following directories:

- **`lua/leyland/plugins/`** - Main plugin configurations
- **`lua/leyland/plugins/lsp/`** - LSP-specific plugins (mason, lspconfig)

## Plugin Loading

lazy.nvim automatically handles plugin loading based on:

- **Event triggers** - Load when specific Neovim events occur
- **File types** - Load only for specific file types
- **Commands** - Load when a command is first used
- **Key mappings** - Load when a keymap is triggered

This ensures fast startup times while maintaining full functionality.
