## Overview

## Telescope

[Telescope](https://github.com/nvim-telescope/telescope.nvim) is a highly extendable fuzzy finder for files, text, Git branches, and more.

### Configuration

Located in `lua/leyland/plugins/telescope.lua:1-45`:

```lua
return {
  "nvim-telescope/telescope.nvim",
  branch = "0.1.x",
  dependencies = {
    "nvim-lua/plenary.nvim",
    { "nvim-telescope/telescope-fzf-native.nvim", build = "make" },
    "nvim-tree/nvim-web-devicons",
    "folke/todo-comments.nvim",
  },
  config = function()
    local telescope = require("telescope")
    local actions = require("telescope.actions")

    telescope.setup({
      defaults = {
        path_display = { "filename_first" },
        mappings = {
          i = {
            ["<C-k>"] = actions.move_selection_previous,
            ["<C-j>"] = actions.move_selection_next,
            ["<C-q>"] = actions.send_selected_to_qflist + actions.open_qflist,
          },
        },
      },
      pickers = {
        live_grep = {
          path_display = { "truncate" },
        },
      },
    })

    telescope.load_extension("fzf")
  end,
}
```

## Harpoon

[Harpoon](https://github.com/ThePrimeagen/harpoon) provides lightning-fast navigation between your most important files.

### Configuration

Located in `lua/leyland/plugins/harpoon.lua:1-42`:

```lua
return {
  "ThePrimeagen/harpoon",
  branch = "harpoon2",
  opts = {
    menu = {
      width = vim.api.nvim_win_get_width(0) - 4,
    },
    settings = {
      save_on_toggle = true,
    },
  },
  keys = function()
    local keys = {
      {
        "<leader>H",
        function()
          require("harpoon"):list():add()
        end,
        desc = "Harpoon File",
      },
      {
        "<leader>h",
        function()
          local harpoon = require("harpoon")
          harpoon.ui:toggle_quick_menu(harpoon:list())
        end,
        desc = "Harpoon Quick Menu",
      },
    }

    for i = 1, 9 do
      table.insert(keys, {
        "<leader>" .. i,
        function()
          require("harpoon"):list():select(i)
        end,
        desc = "Harpoon to File " .. i,
      })
    end
    return keys
  end,
}
```

### Keymaps

| Key | Action | Description |
|-----|--------|-------------|
| `<leader>H` | Add file | Add current file to Harpoon list |
| `<leader>h` | Quick menu | Toggle Harpoon quick menu |
| `<leader>1` | Go to file 1 | Jump to first harpooned file |
| `<leader>2` | Go to file 2 | Jump to second harpooned file |
| `<leader>3` | Go to file 3 | Jump to third harpooned file |
| ... | ... | ... |
| `<leader>9` | Go to file 9 | Jump to ninth harpooned file |

Located in `lua/leyland/plugins/harpoon.lua:12-41`.

### How Harpoon Works

<Steps>
  <Step title="Mark important files">
    While in a file you use frequently, press `<leader>H` to add it to your Harpoon list.
  </Step>
  
  <Step title="Build your list">
    Add up to 9 files that you switch between regularly.
  </Step>
  
  <Step title="Jump instantly">
    Press `<leader>1` through `<leader>9` to jump to files instantly - no fuzzy finding needed!
  </Step>
  
  <Step title="Manage your list">
    Press `<leader>h` to see and reorder your harpooned files.
  </Step>
</Steps>

### Workflow Example

Imagine working on a web app:

1. `<leader>H` in `src/index.tsx` (mark as position 1)
2. `<leader>H` in `src/api/users.ts` (mark as position 2)
3. `<leader>H` in `src/components/Header.tsx` (mark as position 3)
4. `<leader>H` in `src/styles/global.css` (mark as position 4)

Now you can:
- `<leader>1` - Jump to index
- `<leader>2` - Jump to users API
- `<leader>3` - Jump to Header component
- `<leader>4` - Jump to styles

No fuzzy finding, no searching - instant navigation!

## nvim-tree

[nvim-tree](https://github.com/nvim-tree/nvim-tree.lua) is a file explorer sidebar for visual project navigation.

### Configuration

Located in `lua/leyland/plugins/nvim-tree.lua:1-62`:

```lua
return {
  "nvim-tree/nvim-tree.lua",
  dependencies = "nvim-tree/nvim-web-devicons",
  keys = {
    { "<leader>e", "<cmd>NvimTreeFindFileToggle<cr>", desc = "Toggle file explorer" },
  },
  config = function()
    local nvimtree = require("nvim-tree")
    local api = require("nvim-tree.api")

    vim.g.loaded_netrw = 1
    vim.g.loaded_netrwPlugin = 1

    nvimtree.setup({
      view = {
        width = 50,
        relativenumber = true,
        side = "right",
      },
      renderer = {
        indent_markers = {
          enable = true,
        },
        icons = {
          glyphs = {
            folder = {
              arrow_closed = "",
              arrow_open = "",
            },
          },
        },
      },
      actions = {
        open_file = {
          window_picker = {
            enable = false,
          },
        },
      },
      filters = {
        custom = { ".DS_Store" },
      },
      git = {
        ignore = false,
      },
    })
  end,
}
```

    3. Press `<CR>`

  <Step title="Delete file">
    1. Navigate to file
    2. Press `d`
    3. Confirm deletion
  </Step>

    1. Navigate to file
    2. Press `r`
    3. Edit name
    4. Press `<CR>`
  </Step>
  
  <Step title="Move file">
    1. Navigate to file
    2. Press `x` (cut)
    3. Navigate to destination
    4. Press `p` (paste)
  </Step>

## vim-tmux-navigator

[vim-tmux-navigator](https://github.com/christoomey/vim-tmux-navigator) provides seamless navigation between Vim splits and tmux panes.

### Configuration

Located in `lua/leyland/plugins/init.lua:3`:

```lua
"christoomey/vim-tmux-navigator"
```

### Keymaps

| Key | Action | Description |
|-----|--------|-------------|
| `<C-h>` | Navigate left | Move to left split/pane |
| `<C-j>` | Navigate down | Move to lower split/pane |
| `<C-k>` | Navigate up | Move to upper split/pane |
| `<C-l>` | Navigate right | Move to right split/pane |

### Setup with tmux

Add to your `~/.tmux.conf`:

```bash
# Smart pane switching with awareness of Vim splits
is_vim="ps -o state= -o comm= -t '#{pane_tty}' \
    | grep -iqE '^[^TXZ ]+ +(\\S+\\/)?g?(view|n?vim?x?)(diff)?$'"
bind-key -n 'C-h' if-shell "$is_vim" 'send-keys C-h'  'select-pane -L'
bind-key -n 'C-j' if-shell "$is_vim" 'send-keys C-j'  'select-pane -D'
bind-key -n 'C-k' if-shell "$is_vim" 'send-keys C-k'  'select-pane -U'
bind-key -n 'C-l' if-shell "$is_vim" 'send-keys C-l'  'select-pane -R'
```

## Tips

<Tip>
**File Explorer Position:** nvim-tree opens on the right to keep your main code window stable on the left.
</Tip>

<Tip>
**Harpoon Discipline:** Only harpoon files you're actively working on. Remove files when done with them to keep your list focused.
</Tip>

<Tip>
**Telescope Speed:** Press `<Esc>` to enter normal mode in Telescope for faster navigation with `j`/`k`.
</Tip>

<Warning>
**Case Sensitivity:** Telescope searches are case-insensitive by default. Type uppercase letters for case-sensitive search.
</Warning>

<Tip>
**Quick File Tree:** Use `<leader>e` to quickly jump to file explorer, orient yourself, then `<leader>e` again to return to editing.
</Tip>
