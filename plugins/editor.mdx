## Overview

nvim-luffy includes powerful editor enhancement plugins that provide modern IDE features like syntax highlighting, code formatting, automatic pairing, and fast navigation.

## nvim-treesitter

[nvim-treesitter](https://github.com/nvim-treesitter/nvim-treesitter) provides advanced syntax highlighting and code understanding using tree-sitter parsers.

### Configuration

Located in `lua/leyland/plugins/treesitter.lua:1-33`:

```lua
return {
  "nvim-treesitter/nvim-treesitter",
  branch = "master",
  event = { "BufReadPre", "BufNewFile" },
  build = ":TSUpdate",
  config = function()
    require("nvim-treesitter").setup({
      ensure_installed = {
        "json",
        "javascript",
        "typescript",
        "tsx",
        "yaml",
        "html",
        "css",
        "prisma",
        "markdown",
        "markdown_inline",
        "svelte",
        "graphql",
        "bash",
        "lua",
        "vim",
        "dockerfile",
        "gitignore",
        "query",
        "vimdoc",
        "c",
      },
    })
    vim.treesitter.language.register("bash", "zsh")
  end,
}
```

## nvim-autopairs

[nvim-autopairs](https://github.com/windwp/nvim-autopairs) automatically closes brackets, quotes, and other paired characters.

### Configuration

Located in `lua/leyland/plugins/autopairs.lua:1-30`:

```lua
return {
  "windwp/nvim-autopairs",
  event = { "InsertEnter" },
  dependencies = {
    "hrsh7th/nvim-cmp",
  },
  config = function()
    local autopairs = require("nvim-autopairs")

    autopairs.setup({
      check_ts = true, -- enable treesitter
      ts_config = {
        lua = { "string" },
        javascript = { "template_string" },
        java = false,
      },
    })

    -- Integration with nvim-cmp
    local cmp_autopairs = require("nvim-autopairs.completion.cmp")
    local cmp = require("cmp")
    cmp.event:on("confirm_done", cmp_autopairs.on_confirm_done())
  end,
}
```

### Features

- **Smart pairing** - Automatically inserts closing brackets, quotes, etc.
- **Treesitter integration** - Context-aware pairing (e.g., no pairing inside strings)
- **Completion integration** - Works seamlessly with nvim-cmp
- **Language-specific rules** - Different behavior for different languages

## flash.nvim

[flash.nvim](https://github.com/folke/flash.nvim) provides lightning-fast navigation with highlighted jump points.

### Configuration

Located in `lua/leyland/plugins/flash.lua:1-36`:

```lua
return {
  "folke/flash.nvim",
  event = "VeryLazy",
  opts = {
    modes = {
      char = {
        char_actions = function(motion)
          return {
            [";"] = "next",
            [","] = "prev",
          }
        end,
      },
    },
  },
}
```

### Usage

1. Press `s` in normal mode
2. Type the first character(s) of your target
3. Flash highlights all matches with unique labels
4. Type the label to jump instantly

### Character Motion

- `;` - Jump to next match
- `,` - Jump to previous match

## conform.nvim

[conform.nvim](https://github.com/stevearc/conform.nvim) provides fast, asynchronous code formatting with multiple formatter support.

### Configuration

Located in `lua/leyland/plugins/formatting.lua:1-39`:

```lua
return {
  "stevearc/conform.nvim",
  event = { "BufReadPre", "BufNewFile" },
  config = function()
    local conform = require("conform")

    conform.setup({
      formatters_by_ft = {
        javascript = { "prettier" },
        typescript = { "prettier" },
        javascriptreact = { "prettier" },
        typescriptreact = { "prettier" },
        svelte = { "prettier" },
        css = { "prettier" },
        html = { "prettier" },
        json = { "prettier" },
        yaml = { "prettier" },
        markdown = { "prettier" },
        graphql = { "prettier" },
        liquid = { "prettier" },
        lua = { "stylua" },
        python = { "isort", "black" },
      },
      format_on_save = {
        lsp_fallback = true,
        async = false,
        timeout_ms = 3000,
      },
    })
  end,
}
```

### Features

- **Format on save** - Automatically format files when saving
- **LSP fallback** - Use LSP formatter if conform formatter unavailable
- **Multiple formatters** - Chain formatters (e.g., isort then black)
- **Visual mode** - Format selected ranges only
- **Fast execution** - Async formatting with 3-second timeout

### Manual Formatting

Format manually with `<leader>mp`:

```vim
" Format entire file
<leader>mp

" Format selection (in visual mode)
V
<select lines>
<leader>mp
```

## Integration

All editor plugins work together seamlessly:

- **Treesitter + Autopairs** - Context-aware bracket insertion
- **Flash + Treesitter** - Syntax-aware navigation
- **Conform + LSP** - Formatting with fallback to language servers
- **Autopairs + nvim-cmp** - Smart pairing during completion
